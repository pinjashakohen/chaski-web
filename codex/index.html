The Problem

You finished the work. Can you prove when?

Manuscripts, translations, teachings — you pour months into them, then have no simple record that this exact version existed on this exact date, under your name.

no proof of authorship no timestamp "which draft was final?" silent edits nothing to point to
What Codex Does

A fingerprint for finished work

Everything below runs in the app today. Where something is architecture-for-later, we say so plainly — no on-chain claims that aren't real yet.

01 · FINGERPRINT

SHA-256 hash

Paste a title and the finished text. The server computes a SHA-256 content hash — a fixed signature unique to that exact document.

02 · STAMP

Immutable timestamp

Each registration records the hash algorithm and a permanent created_at time — a fixed record of when this version was sealed.

03 · TRACK

Status you can see

Registrations move through pending → verified → published, shown as a clear badge so you always know where a work stands.

04 · READY

Chain-ready, honestly

The record carries empty fields for an IPFS reference and a chain transaction — reserved for future anchoring. Today it's SHA-256 + timestamp; no blockchain write happens yet.

Honest status

Live for hashing. Not yet on-chain.

Live now Codex is shipped at app.pinjasemet.com/home/codex. SHA-256 fingerprinting, timestamping, status tracking and per-account isolation all work today. The IPFS and blockchain fields exist in the record but are placeholders — no on-chain registration happens yet, and we won't pretend otherwise. It's "verifiable SHA-256 today, blockchain-ready architecture."
